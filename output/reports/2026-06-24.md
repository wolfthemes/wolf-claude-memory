## Insights

1. **Le lancement du 1er juillet est l'opportunité principale.** C'est dans 7 jours. Tout le contenu de la semaine prochaine doit soutenir et amplifier ce moment.

2. **Aucune donnée d'engagement Buffer disponible.** Les métriques de reach/likes ne sont pas encore remontées via l'API (le paramètre `startDateTime` est requis avec un format différent). À corriger pour le prochain rapport.

3. **42 thèmes n'ont jamais été postés.** Avec une cadence de 1/semaine, il faut ~10 mois pour couvrir tout le catalogue — il faut soit accélérer la cadence, soit prioriser stratégiquement (bestsellers, niches à fort hashtag, thèmes à fort potentiel visuel).

4. **Instagram est le canal principal mais sous-exploité.** Les posts sont multi-canaux mais probablement pas optimisés par plateforme (même texte partout).

5. **Aucune présence Pinterest.** Marqué "pending" dans le stack — Pinterest est pourtant la 2ème priorité pour les thèmes WordPress (trafic organique long terme très fort dans cette niche).

## Recommandations pour la semaine du 25 juin – 1er juillet

### Priorité 1 — Lancement store (1er juillet)
- Préparer 2-3 posts de lancement pour le 1er juillet : un pour Instagram (visuel fort), un pour X (urgence/annonce), un pour Facebook (détails/CTA clair)
- Créer un post "before/after" : ThemeForest vs achat direct — ce que le client gagne

### Priorité 2 — Activer Pinterest
- Créer les tableaux Pinterest : "Music WordPress Themes", "Fashion WordPress Themes", etc.
- Premier post : Seijaku ou Loud (visuellement forts, bons hashtags)

### Priorité 3 — Cadence post-lancement
- Après le 1er juillet, passer à 2 posts/semaine pour accélérer la couverture du catalogue
- Alterner : product showcase (thème) + contenu founder/BTS
