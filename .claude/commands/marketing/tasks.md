# /marketing-tasks

Analyse les données de performance et suggère des tâches marketing actionnables.

## Étapes

### 1. Lis le contexte disponible

- Dernier rapport Analytics : `output/reports/` (le fichier le plus récent)
- Log de contenu : `output/content-log.md`
- Tâches en cours : `TASKS.md`

### 2. Identifie les opportunités

En croisant les données, cherche :

**Opportunités de contenu**
- Thèmes à fort trafic GA4 mais pas encore postés sur les réseaux
- Catégories sous-représentées dans le contenu publié
- Posts Buffer qui ont sur-performé → peut-on créer du contenu similaire ?

**Optimisations techniques**
- Pages de thèmes avec fort trafic mais taux de rebond élevé → améliorer la page produit
- Sources de trafic inattendues → creuser et amplifier

**Opportunités réseau**
- Si Instagram sur-performe → augmenter la fréquence
- Si Pinterest génère peu → tester un autre format (infographie, comparatif)

**Saisonnalité**
- Quelle est la date ? Y a-t-il une opportunité saisonnière dans les 4 prochaines semaines ?
  (Ex : rentrée → thèmes portfolio, Black Friday → promos, été → thèmes lifestyle)

### 3. Génère les tâches

Rédige des tâches actionnables dans ce format :
- Chaque tâche commence par un verbe d'action
- Inclut pourquoi (basé sur les données)
- Estime l'effort : [Rapide / Moyen / Long]

### 4. Met à jour TASKS.md

Ajoute les nouvelles tâches dans `TASKS.md` sous la section **Active**, en évitant les doublons avec les tâches existantes.

Format à utiliser :
```
- [ ] {Tâche} — *raison basée sur données* [Rapide/Moyen/Long]
```

Ne supprime pas les tâches existantes. Ajoute seulement.
